# TEST_CHECKLIST.md — 体验测试清单

> 三条主干：**A 用户线** / **B 管理员线** / **C 安全线**。建议用两个微信号分别跑用户线和管理员线。
> 准备：① init 已跑、集合有数据；② 全部云函数已部署；③ 集合安全规则已配（见 SECURITY.md）。

## A 用户线（普通会员 / 消费者）

### A1 首次进入 & 入会礼
- [ ] 新微信号首次进入 → 自动注册，无需手动登录
- [ ] 首登积分 = **120**（新客入会礼到账）
- [ ] 积分明细里有一条「入会 +120」
- [ ] 老用户再次进入 → **不**重复送 120
- [ ] 断网/弱网进入 → 不白屏卡死（页面能加载，数据稍后补上）

### A2 首页（网购式）
- [ ] 顶部显示「是模玩店！」字标
- [ ] 轮播 banner 自动轮播、圆点指示
- [ ] 5 个图标导航可点（兑换/会员码/兑换记录/签到/上新）
- [ ] 商品瀑布流双列显示，价格为红色「N 积分」
- [ ] 无 banners 时显示占位、无商品时显示空状态文案

### A3 每日签到
- [ ] 当天首次签到 → 弹「签到 +10 积分」，积分增加
- [ ] 当天再签 → 提示「今天已经签到过啦」，积分不变
- [ ] 签到后积分明细多一条「签到 +10」

### A4 积分兑换（核心 + 分支）
- [ ] 兑换页顶部显示「我的积分」
- [ ] 分类切换（全部/优惠券/周边/贴纸/盲盒）筛选正确
- [ ] **积分足够** → 确认兑换 → 弹兑换码 → 积分扣减、库存 -1
- [ ] **积分不足** → 点兑换 → 提示「积分不足」，不扣分
- [ ] **库存为 0** → 按钮显示「已兑完」且置灰，点不动
- [ ] 兑换成功后积分明细多一条「兑换 -N」
- [ ] 兑换成功后「我的兑换」多一单，状态「待核销」

### A5 我的兑换 / 券码
- [ ] 兑换记录列表显示商品图/名/单号/积分/日期
- [ ] 状态筛选（全部/待核销/已完成/已取消）正确
- [ ] 待核销单可「查看券码」弹出兑换码

### A6 我的积分 & 明细
- [ ] 积分页显示当前积分、成长值、等级
- [ ] 成长进度条 + 「再赚 N 升级 X」正确
- [ ] 明细按时间倒序，获取为红色 +，消费为灰色
- [ ] 明细筛选（全部/获取/消费）正确

### A7 会员码
- [ ] 显示会员号（= users._id）
- [ ] 点会员号可复制
- [ ] 显示头像 + 昵称 + 等级

### A8 资料编辑（曾经的坑，重点测）
- [ ] 改昵称 → 失焦 → 弹「已保存」，**刷新/切页后不回退成"新会员"**
- [ ] 改头像 → 选图 → 「上传头像」→ 显示新头像，**换页后仍在**
- [ ] 头像右下角有红色可点角标 + 「点击头像/昵称可修改」提示
- [ ] 空昵称不触发保存

### A9 商品更新页
- [ ] 动态列表显示（新品到货/活动预告/店铺公告）
- [ ] 顶部 tab 筛选正确

## B 管理员线

### B1 成为管理员（OTP，重点测分支）
- [ ] 普通用户「我的」页**看不到**任何员工/管理入口（只有底部不起眼的版本号）
- [ ] 连点底部「模玩店 v1.0.0」**7 下** → 进员工入口（吉祥物 + 输入框）
- [ ] `node tools/otp.js` 取码 → 30 秒内输入 → 弹「已成为管理员」
- [ ] **输错码** → 弹「动态码错误或已过期」+ 服务器时间
- [ ] **过期码**（拖到 90 秒后再输）→ 失败
- [ ] 成为管理员后，「我的」页出现「店铺管理」

### B2 管理员「我的」页
- [ ] 身份标签显示「管理员」
- [ ] 只显示：店铺管理 / 店员工作台 / 退出员工身份
- [ ] **不**显示：积分卡 / 我的权益 / 积分明细等会员功能

### B3 店铺管理 — 商品
- [ ] 新增商品（名/分类/积分/库存/上架开关/图）→ 列表出现
- [ ] **上传商品图** → 「图片已上传」→ 表单预览显示（失败会弹真实错误）
- [ ] 编辑已有商品 → 改动生效
- [ ] 删除商品 → 列表移除
- [ ] 上架开关关闭（下架）→ 用户端首页/兑换页**不**显示该商品
- [ ] 新增的上架商品 → 用户端兑换页**能**看到并兑换

### B4 店铺管理 — 轮播 / 动态
- [ ] 新增轮播（标题/排序/图）→ 用户端首页 swiper 出现
- [ ] 删除轮播 → 首页不再显示
- [ ] 新增动态 → 用户端「商品更新」页出现
- [ ] 删除动态 → 不再显示

### B5 店员管理（不碰数据库加店员）
- [ ] 输入对方**会员号** + 角色开关关 → 对方成为「店员」
- [ ] 角色开关开 → 对方成为「管理员」
- [ ] 会员号不存在 → 提示「会员不存在」
- [ ] 删除某店员 → 列表移除，对方失去权限
- [ ] 删自己 → 提示「不能移除自己」

### B6 店员工作台（核心收银台）
- [ ] 非店员进入 → 显示「无权限」拦截
- [ ] 输入/扫会员号 → 查询出会员（昵称/等级/积分）
- [ ] **加积分** → 对方积分增加、流水加一条、最近操作出现
- [ ] **扣积分** → 正常扣减
- [ ] 扣超过余额 → 提示「积分不足，无法扣减」
- [ ] **核销兑换码** → 该兑换单变「已完成」
- [ ] **重复核销**同一码 → 提示「该券已核销」
- [ ] 不存在的兑换码 → 提示「兑换码不存在」

### B7 退出 & 复登
- [ ] 「退出员工身份」→ 确认 → 不再是管理员
- [ ] 退出后「我的」页恢复成普通会员视图
- [ ] 退出后再 OTP 登录 → 能重新成为管理员

## C 安全线（详见 SECURITY_TEST.md，这里抽测关键 4 条）

- [ ] 调试器 Console 执行 `wx.cloud.database().collection('users').doc('x').update({data:{points:99999}})` → **permission denied**
- [ ] 非店员账号调 `addPoints` / `getMember` → 返回「无权限」
- [ ] 非管理员调 `admin`（list goods）→ 返回「无权限」
- [ ] `updateProfile` 传 `{nickName:'a', points:9999}` → 只改昵称，积分不变
- [ ] 普通用户连点版本号也只到员工入口，**没有 OTP 进不去**管理后台
