# 「是模玩店！」小程序 — 上线操作清单

> 按顺序勾选即可上线。🔴=阻断上线必做，🟡=强烈建议，⚪=可上线后补。
> 费用：唯一硬性年费是**微信认证 ¥300/年**；云开发初期 ¥0（最多约 ¥240/年，甲方付）。

---

## 一、主体与认证 🔴

- [ ] 准备资料：营业执照、法人身份证、法人微信号+手机号、结算银行卡（若开支付）
- [ ] 微信公众平台 → 设置 → **微信认证**（¥300/年，每年续）。个体户可走「法人微信扫脸验证」，免对公打款
- [ ] 补全小程序基本信息：名称「是模玩店！」、Logo、简介
- [ ] 选择**服务类目**（零售/周边相关，营业执照经营范围要对得上）

## 二、隐私与合规 🔴

- [ ] 公众平台 → 设置 → **用户隐私保护指引**：勾选「微信昵称、头像」等收集项并提交（不配会被拒审）
- [ ] 填真实信息：`pages/rules/rules.js` 的 `shopName`（营业执照店名）、`contact`（真实客服电话）
- [ ] 规则页《积分会员服务规则》已就绪（积分获取/有效期/核销/免责/最终解释权），确认内容无误

## 三、云端部署 🔴

- [ ] **上传部署全部云函数**（右键 → 上传并部署，云端安装依赖）：
  - `login`、`init`、`signIn`、`addPoints`、`redeemGoods`、`verifyRedeem`、`updateProfile`、`getMember`
  - `claimAdmin`、`resignStaff`、`admin`
  - 🆕 `staffSecret`、`getAudit`
- [ ] `claimAdmin` 云函数 → 配置 → **环境变量** `TOTP_SECRET` 填入本地 `tools/.totp_secret` 里那串（主口令密钥）
- [ ] 跑一次 `init`（在小程序「员工入口」或云端测试调用）→ 建齐集合，含新增 `audit_log`
- [ ] 云存储 → **存储权限** 选「**所有用户可读，仅创建者可读写**」（第 1 个，图片才能上传+显示）
- [ ] 数据库 → 各集合**安全规则**：
  - `goods` / `banners` / `updates`：所有人可读、仅管理端可写
  - `users` / `staff` / `points_log` / `redeems` / `sec_lock` / `sec_otp` / **`audit_log`**：**仅管理端**（客户端全锁，数据都走云函数）

## 四、首个管理员 🔴

- [ ] 用验证器 App（Google/微软/腾讯）添加主口令密钥（`tools/.totp_secret`，或 `node tools/otp.js` 出码）
- [ ] 小程序「我的」页连点版本号 → 进「员工入口」→ 输主口令动态码 → 成为店长（系统自动发个人密钥，记得扫码保存）
- [ ] 在「店铺管理 → 店员」里添加店员 → 点「生成码」→ 让店员用验证器扫码 → 店员到员工入口输码登录（7 天有效）

## 五、真机自测 🟡

- [ ] 按 `TEST_CHECKLIST.md` 跑用户线 / 管理员线 / 安全线
- [ ] 重点验证：会员码扫码加分、按消费额加分、积分抵现（余额不足要拦）、核销、操作日志有记录

## 六、上线 🔴

- [ ] 开发者工具「上传」→ 填版本号 → 公众平台「版本管理」设为**体验版** → 加体验成员自测
- [ ] 体验无误 → **提交审核**（填好功能页面、测试说明；如审核要测试账号，提供店员动态码登录方式说明）
- [ ] 审核通过 → **发布**
